# CIOS KERNEL | Clinical Intelligence OS
## Enterprise Geriatric Monitoring & Artificial Intelligence

CIOS is a centralized clinical intelligence platform custom-engineered for **Funmilola Home for the Aged, Ogbomosho**. It serves as an AI-orchestrated middleware layer designed to bridge geriatric home management with hyper-local clinical intelligence.

### 🌐 Localization: Yoruba & English
Built for the specific cultural context of Ogbomosho, Nigeria, CIOS features a seamless toggle between English and Yoruba (Kẹ́nẹ́lì), ensuring that clinical personnel and support staff can interact with the system in their primary language without losing technical precision.

---

## 🚀 Core Features

### 1. CIOS Kernel (Intelligence Dashboard)
*   **Infrastructure Monitoring**: Real-time visualization of health vectors across geriatric wards.
*   **Critical Monitor**: Automated detection of high-risk service user profiles using the Zero-Law oversight protocol.
*   **Operational Metrics**: Population tracking, domain status (Residential, Palliative, Neuro-Rehab), and AI precision confidence intervals.

### 2. Ward Management (Service User Registry)
*   **Filtered Observability**: Segregation of ward data across Residential, Complex Care, and Home Care environments.
*   **Condition Tagging**: Real-time status indicators (Hypertension, Mobility Risk, Cognitive Decline) and emergency warning states.
*   **Ogbomosho-Node Syncing**: Data persistence localized to the specific facility node for low-latency retrieval.

### 3. CIOS Advisory (Intelligence Module)
*   **Multi-Modal Consultations**: AI-powered clinical advisory for service user care, mentorship, and examiner workflows.
*   **Zero-Law Protocol Enforcement**: A built-in ethical safety layer that ensures all AI output is framed as secondary to human physician judgment.
*   **Localized Context**: The engine understands local health paradigms and translates clinical complexity into actionable insights in both English and Yoruba.

### 4. Team Matrix (Medical Personnel)
*   **Credentialed Registry**: Centralized view of authorized clinical staff ranging from Neuro-Geriatricians to Registered Nurses.
*   **Duty Status**: Visualization of primary ward assignments and clinician impact metrics.

### 5. System Configuration (Cognitive Matrix)
*   **AI Cascade Routing**: Support for multiple intelligence providers including Google Gemini, Groq, OpenRouter, and local Ollama nodes.
*   **Zero-Knowledge Headers**: Security-hardened request routing for privacy compliance in medical environments.

## 🛠 Technical Architecture

*   **Runtime**: React 18 + Vite (TypeScript)
*   **Styling**: Atomic Tailwind CSS with a "Brutalist-Clinical" aesthetic (Deep Ink, Glass-morphism, and Toxic Green accents).
*   **Animations**: Motion (Framer Motion) for staggered lifecycle transitions.
*   **Intelligence Engine**: Google Gemini Pro (Primary) with a dynamic fallback cascade.
*   **Localization Strategy**: Static JSON Dictionary with dynamic state binding for EN/YO transitions.

## ⚖️ The Zero-Law Oversight Nuance
In an enterprise healthcare setting, AI hallucinations are not just bugs; they are safety risks. CIOS implements the **Zero-Law Protocol**:
1.  **Doctor-in-the-Loop**: All AI-driven advice requires clinician validation before ward execution.
2.  **Explicit Warning States**: The UI highlights AI uncertainty through "Kernel Fluctuations" and precision percentage alerts.
3.  **Language Parity**: Yoruba translations are medically verified to ensure no clinical nuances are lost in the vernacular.

## 📦 Getting Started

### Prerequisites
*   Node.js 18+
*   Gemini API Key (configured in AI Studio Settings or `.env`)

### Installation
1.  Install dependencies:
    ```bash
    npm install
    ```
2.  Start the development server:
    ```bash
    npm run dev
    ```
